About the experiment

This is another experiment clubbing remote data with local data. The reviews fetched from the remote database will have non local reviewers and the web service will not reaveal these users information as it is confidential. But the local reviews will have local reviewers whose information is availbale in the mongoDB. When adding a local review to the database a flag is added to distinguish it from the remote reviews while rendering. So only the local reviewers will have link to follow through otherwise no link is available. On clicking the reviewer's name, we will be redirected to his profile where we can see all his favorite products. The following schema is followed to add a local review which has a local flag :

Code Snippet:

// creates a new review object adding all necessary fields var myReview = {
comment:review.comment,
title:review.title,
submissionTime : new Date(),
reviewer : reviewer,
rating: review.rating,
local: true, // tells if the review is a local one
sku:sku // id of the product for which the review has been written
};

While rendering the reviews locally, 'local' flag is used to decide whether a link should be present.

Code Snippet:

<span ng-if="review.local">
<a href="#/showUserDetails/{{review.reviewer[0].name}}">{{review.reviewer[0].name}}</a>
</span>
<span ng-if="!review.local">
{{review.reviewer[0].name}}
</span>

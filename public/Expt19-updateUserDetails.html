About the experiment

This experiment is about letting the user to change his password. Though this experiment appears to be simple, it is not so. Because when user tries to change his password, he must enter his old password for security. I have written an angular directive which checks if the entered password matches the current one in the database. Based on the value returned from the databse, an error will be thrown. Also separate directive is available to check if the new passwords entered match each other. Otherwise error will be thrown. The submit button will be enabled only if there is no error. The directive that checks the password match is as follows :

Code snippet : 
		app.directive('checkPassword', ['$http', function($http, $timeout) {  
		return {
		require: 'ngModel',
		link: function(scope, elem, attrs, ctrl) {
			scope.busy = false;
			scope.$watch(attrs.ngModel, function(value) {
				$http.post('/user/checkPassword',{username:scope.user.username, password:value})
				.success(function(data) {
					scope.busy = false;

				})
				.error(function(data) {
					if (data.noMatch) {
						ctrl.$setValidity('noMatch', true);
					}
					scope.busy = false;
				});
			})
		}
